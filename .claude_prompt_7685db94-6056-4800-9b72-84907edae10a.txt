**Title**:
[PR Feedback] Span Naming Convention

**Description**:
Based on consolidated PR review, address the following in **Span Naming Convention**:

- **FR-6/FR-9: Span names do not embed adapter ID or domain as specified**: FR-6 states each adapter fetch span must be 'named with the adapter's adapter_id', and FR-9 gives examples (`adapter.fetch apple_reminders`, `domain.chunk tasks`). The implementation uses generic names (`adapter.fetch`, `domain.chunk`) with adapter ID in span attributes only. This deviates from the spec and affects pre-built SigNoz filter/group-by expressions that rely on span name patterns.
- **FR-8/US-7: Scheduler root span names differ from spec**: US-7 specifies root spans `scheduler.poll` and `scheduler.watch`. The implementation uses `scheduler.poll.tick`, `scheduler.poll.source`, `scheduler.watch.event`, and `scheduler.watch.retry_flush`. While more granular, this breaks SigNoz filter expressions referencing the agreed-upon names.
