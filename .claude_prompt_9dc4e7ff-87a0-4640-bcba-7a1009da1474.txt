Phase 4: Search and browser view component replacements

**Description**:
Replace custom component patterns in the search and browser views: domain filter chips, the search result detail panel, view-mode tab controls, and raw select elements inside the data table.

## Requirements
- Replace the custom `DomainChip` toggle buttons in `search.tsx` (lines 52–83) with Heimdall `Chip` using `className` overrides referencing existing `--domain-*` CSS tokens for per-domain color identity (FR-3.1, ADR 5)
- Replace inline domain tag rendering in `index.tsx` (dashboard) and `notes.tsx` with Heimdall `Chip` (FR-3.2)
- Replace the `DetailPanel` component in `search.tsx` (lines 236–354) with Heimdall `Drawer` at `position="right"`, preserving all chunk content: title, source, domain, content text, and metadata (FR-6.1)
- Ensure `Drawer` closes on Escape key and backdrop click (FR-6.1 acceptance criteria)
- Replace view-mode toggle buttons in `search.tsx` with Heimdall `TabBar` (FR-5.2)
- Replace the Sources/Chunks/Versions button group in `browser.tsx` with Heimdall `TabBar` using `tabs` prop with `id`, `label`, and optional `count` (FR-5.1)
- Replace raw `<select>` elements for facet filters (DataTable.tsx lines 327–340) and page-size selection (lines 436–446) with Heimdall `Select` (FR-4.1, FR-4.2)
- Wire search error states (query failure, empty result from network error) to `useToast()` (FR-8.3)

## Design Guidance
Implements the Phase 4 Search + Browser View steps from the architecture design. ADR 5 defines the domain chip color strategy (Chip + className overrides). The architecture document notes the `Drawer` content migration from `search.tsx:236-354` and the `TabBar` tab array shape for `browser.tsx`.
