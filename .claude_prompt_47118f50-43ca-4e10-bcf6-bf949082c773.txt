**Title**:
Phase 3: Apple Podcasts adapter

**Description**:
Implement `ApplePodcastsAdapter` as a dual-endpoint, dual-domain adapter. Listen history items are `Domain.EVENTS`; transcript items are `Domain.DOCUMENTS` via per-item domain override. This adapter is high-value because transcript search enables full-text retrieval of episode content.

## Requirements
- Create `adapters/apple_podcasts.py` as a dual-endpoint adapter with multi-endpoint error isolation
- Adapter declares `domain = Domain.EVENTS`; transcript items override `NormalizedContent.domain = Domain.DOCUMENTS`
- **Listen history** (`GET /podcasts/listen-history?since=`):
  - `source_id: podcasts/listen/{id}`
  - `EventMetadata.title = "{showTitle} — {episodeTitle}"`
  - `EventMetadata.start_date = listenedAt`
  - `EventMetadata.source_type = "podcast_listen"`
  - `durationSeconds`, `playedSeconds`, `completed`, `episodeGuid`, `feedUrl` → `extra_metadata`
- **Transcripts** (`GET /podcasts/transcripts?since=`):
  - `source_id: podcasts/transcript/{id}`
  - `DocumentMetadata.title = episodeTitle`
  - `DocumentMetadata.author = showTitle`
  - `NormalizedContent.markdown = transcript`
  - `DocumentMetadata.created_at = transcriptCreatedAt`
  - `DocumentMetadata.modified_at = publishedDate`
  - `DocumentMetadata.source_type = "podcast_transcript"`, `document_type = "text/plain"`
  - `transcriptSource`, `episodeGuid`, `durationSeconds`, `playStateTs` → `extra_metadata`
- Each endpoint is fetched independently; one endpoint failure yields `PartialFetchError`, not total failure
- Empty transcript text produces no yielded content
- Register in `adapters/__init__.py` under `httpx` guard

## Design Guidance
Implements Phase 3 (Podcasts Adapter) from the architecture design. Uses Pattern 2 (Multi-Endpoint Adapter with Partial Failure), Pattern 3 (Per-Item Domain Override), and Pattern 4 (Metadata via `extra_metadata`). ADR-3 documents the dual-domain rationale.

## Acceptance Criteria
- [ ] `ApplePodcastsAdapter` is importable from `context_library.adapters` when `httpx` is installed
- [ ] Listen history happy path: correct `EventMetadata` with title formatted as `"{showTitle} — {episodeTitle}"`
- [ ] Transcript happy path: correct `DocumentMetadata` with `domain=DOCUMENTS` override, full text in markdown
- [ ] Partial failure test: transcripts endpoint down → listen history still yielded, `PartialFetchError` raised
- [ ] Total failure test: both endpoints down → `AllEndpointsFailedError` raised
- [ ] Empty transcript body → no content yielded for that item
