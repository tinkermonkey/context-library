**Title**:
Phase 5: Resizable split-panel layout upgrades

**Description**:
Replace fixed CSS grid layouts in the notes, messages, and file browser views with Heimdall `SplitPane` components so users can resize panels to suit their content.

## Requirements
- Replace the three-column fixed CSS grid in `notes.tsx` with two nested `SplitPane` components: outer splits vault tree (~20%) from the rest (~80%); inner splits note list (~35%) from note content (~65%) (FR-10.1, ADR 3)
- Replace the two-column fixed CSS grid in `messages.tsx` with a single `SplitPane` splitting thread list from conversation (FR-10.2)
- Replace the fixed CSS grid split in the file browser view with a `SplitPane` splitting the file tree from the content area (FR-10.3)
- Set `direction="horizontal"` and apply `minSize` constraints on all `SplitPane` instances to prevent panels collapsing to zero
- All content within panes must remain fully visible and usable after resizing

## Design Guidance
Implements the Phase 5 Layout Upgrades steps from the architecture design. ADR 3 defines the nested SplitPane approach for the three-panel notes layout. The architecture document specifies initial split percentages and the `first`/`second` ReactNode props used for nesting.

## Acceptance Criteria
- [ ] Notes view displays vault tree, note list, and note content in three resizable panels
- [ ] Messages view displays thread list and conversation in two resizable panels
- [ ] File browser displays file tree and content area in two resizable panels
- [ ] Dragging any split handle resizes both panes proportionally
- [ ] No panel collapses to zero width at its minimum size constraint
- [ ] All content within resized panes remains fully visible and usable
