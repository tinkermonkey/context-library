[PR Feedback] Telemetry Configuration & Dependencies

**Description**:
Based on consolidated PR review, address the following in **Telemetry Configuration & Dependencies**:

- **Module-level `opentelemetry` imports rely on transitive dependency**: `pipeline.py`, `embedder.py`, `entity_linker.py`, `query.py`, and `poller.py` all import `from opentelemetry import trace` at module level unconditionally. `opentelemetry-api` is not declared as a base dependency in `pyproject.toml` — it only works today because `chromadb` transitively installs it. If chromadb ever drops or makes its OTel dependency optional, all core modules will fail to import.
- **`otlp_protocol` field accepts arbitrary strings with silent fallthrough**: `src/context_library/telemetry/config.py:26` types `otlp_protocol` as `str` with no validation. The consuming code uses `if config.otlp_protocol == "http/protobuf"` with a bare `else` fallback to gRPC. A typo like `"http"` or `"HTTP/PROTOBUF"` silently gets gRPC behavior. Should use `Literal["grpc", "http/protobuf"]` for compile-time and runtime enforcement.
