**Title**:
[PR Feedback] Backend Schema and Query Layer

**Description**:
Based on consolidated PR review, address the following in **Backend Schema and Query Layer**:

- **SQL filter uses `GLOB` instead of spec-prescribed `LIKE`**: The architect spec explicitly prescribes `LIKE ? || '%'` but the implementation uses `GLOB ? || '*'` with character escaping. SQLite `GLOB` is case-sensitive by default while `LIKE` is case-insensitive — this changes filtering behavior for file paths on case-insensitive filesystems (macOS, Windows). Index interaction may also differ.
- **`server/schemas.py` not updated with `source_id_prefix`**: The architect spec calls out adding `source_id_prefix` to `SourceQueryParams` in `src/context_library/server/schemas.py`. The parameter was added to the route handler signature and frontend types, but if `SourceQueryParams` is a Pydantic schema used server-side, it does not reflect the new parameter.
