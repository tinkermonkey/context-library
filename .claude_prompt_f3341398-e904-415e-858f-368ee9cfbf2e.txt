**Title**:
Phase 1: Telemetry foundation — package, configuration, SDK initialization, and log bridge

**Description**:
Establish the telemetry subsystem that all subsequent phases depend on. This phase creates the new `telemetry/` package, defines all configuration via `TelemetryConfig`, initializes TracerProvider and LoggerProvider with OTLP exporters, wires FastAPI and httpx auto-instrumentation, attaches the Python logging bridge, and integrates startup/shutdown into the server lifespan. When this phase is complete, the server exports HTTP endpoint spans and forwards all `context_library.*` log records to SigNoz with trace correlation — without any changes to existing log call sites.

## Requirements
- Create `src/context_library/telemetry/__init__.py` with `setup_telemetry()` and `shutdown_telemetry()`
- Create `src/context_library/telemetry/config.py` with `TelemetryConfig` (pydantic-settings, `CTX_` prefix)
- Config fields: `CTX_OTEL_ENABLED` (default `false`), `CTX_OTLP_ENDPOINT`, `CTX_OTLP_PROTOCOL` (`grpc` or `http/protobuf`), `CTX_OTEL_SERVICE_NAME` (default `context-library`), `CTX_OTEL_ENVIRONMENT` (default `production`)
- OTLP Resource must include `service.name`, `service.version` (from package metadata), `deployment.environment`
- `setup_telemetry()` must be a no-op (return immediately, no providers created, no network connection) when `CTX_OTEL_ENABLED` is falsy or `CTX_OTLP_ENDPOINT` is unset
- Attach `opentelemetry.sdk._logs.LoggingHandler` to the `context_library` root logger; stdout/stderr logging must continue unchanged
- Wire `FastAPIInstrumentor.instrument_app(app)` for HTTP endpoint spans (FR-4)
- Wire `HTTPXClientInstrumentor().instrument()` for outbound adapter HTTP calls with W3C Traceparent propagation (FR-6)
- Call `setup_telemetry()` on lifespan startup and `shutdown_telemetry()` (flush + provider shutdown) on lifespan teardown in `server/app.py`
- Add `otel` optional dependency group to `pyproject.toml` with `opentelemetry-api>=1.20`, `opentelemetry-sdk>=1.20`, both grpc and http/protobuf OTLP exporters, fastapi and httpx instrumentation libraries
- Add `mypy` override `ignore_missing_imports = true` for `opentelemetry.*` in `pyproject.toml`
- Use gRPC exporter as default; HTTP/protobuf exporter when `CTX_OTLP_PROTOCOL=http/protobuf`
- FR-1, FR-2, FR-3, FR-4, FR-6 (auto-instrumentation portion), FR-10, FR-12, FR-13

## Design Guidance
Implements Phase 1 (Foundation) and Phase 6 (OTLP Log Signal) from the architecture design. Follows ADR-1 (programmatic SDK initialization in lifespan), ADR-2 (gRPC default), ADR-3 (CTX_ prefix config), ADR-4 (log bridge pattern), and the new module layout described in 'New Module: src/context_library/telemetry/'. The Component Reuse table in the architecture document identifies the exact integration points in `server/app.py`.
